Gigzito Backend API — Mobile Reference
Base URL (dev): your Replit dev URL
Base URL (production): http://5.78.128.185 (Hillsboro VPS)
Auth: All protected routes accept Authorization: Bearer <token> header
Content-Type: application/json for all POST/PUT/PATCH requests

MOBILE AUTH (JWT — use these, not the web cookie routes)
Method	Path	Auth	Body	Returns
POST	/api/mobile/login	None	{ email, password }	{ mfaRequired: true, email }
POST	/api/mobile/mfa/verify	None	{ email, code }	{ token, user }
POST	/api/mobile/refresh	Bearer	—	{ token, user }
GET	/api/mobile/me	Bearer	—	{ user, profile }
user object: { id, email, role, subscriptionTier }
profile object: { username, displayName, avatarUrl, bio }
Subscription tiers: GZLurker (free) · GZMarketer · GZMarketerPro · GZBusiness · GZEnterprise

ERROR RESPONSES
All errors return { message: string }. Common codes:

Code	Meaning
400	Bad request / validation failed
401	Not authenticated — missing or invalid token
403	Forbidden — wrong tier or role
404	Not found
409	Conflict (e.g. email already registered)
500	Server error
